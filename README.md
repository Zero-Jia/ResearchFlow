# ResearchFlow 初始化说明

## 1. 项目名称
ResearchFlow：基于 LangChain 的 AI 多工具研究助理

## 2. 项目定位
这是一个面向“信息研究、资料整理、结构化总结”的 AI Agent 应用。

用户输入一个研究任务后，系统能够：
- 理解任务意图
- 调用多个工具获取信息
- 对结果进行整理和总结
- 输出结构化研究报告
- 通过 WebSocket 实时展示执行过程
- 使用 LangSmith 观察 Agent 执行链路

## 3. 这个项目为什么是 Agent 项目
它不是简单的问答系统，而是一个能根据任务动态决策、调用工具、多步执行并生成结果的智能体系统。

核心特征：
- 任务理解
- 工具调用
- 多步执行
- 会话记忆
- 结构化输出

## 4. 和普通 Chatbot / RAG 的区别

### 和普通 Chatbot 的区别
普通聊天机器人通常只是“一问一答”，主要依赖模型直接生成答案。
ResearchFlow 会根据任务选择工具，并执行一个完整研究流程。

### 和普通 RAG 的区别
普通 RAG 重点是“检索 + 生成”。
ResearchFlow 不只是检索知识，还会：
- 判断任务类型
- 组合多个工具
- 做对比、总结、规划
- 生成结构化研究报告

## 5. 核心用户场景

### 场景 1：主题研究
示例：
“帮我研究一下 LangChain 中间件的作用，并整理成适合面试复习的总结”

### 场景 2：概念对比
示例：
“比较 LangChain 和 LangGraph 的区别，并告诉我学习顺序”

### 场景 3：学习规划
示例：
“帮我整理 RAG 评测方向的学习路线，并给出阶段性建议”

### 场景 4：个性化研究输出
示例：
“以后优先给我技术化表达，先总结，再列细节”
